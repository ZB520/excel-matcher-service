# 环境变量配置说明

本文档说明如何为 Zeabur 和阿里云函数计算（FC）配置必要的环境变量。

---

## 一、Zeabur 环境变量配置

登录 [Zeabur 控制台](https://zeabur.com)，进入你的项目，在"环境变量"（Environment Variables）中添加以下配置：

### 必需的环境变量

| 环境变量名 | 说明 | 示例值 |
|---|---|---|
| `OSS_ACCESS_KEY_ID` | 阿里云 AccessKey ID | `LTAI5t...`（你的实际 AK） |
| `OSS_ACCESS_KEY_SECRET` | 阿里云 AccessKey Secret | `your_secret_here`（你的实际 SK） |
| `OSS_ENDPOINT` | 阿里云 OSS 访问域名 | `https://oss-cn-hangzhou.aliyuncs.com` |
| `OSS_BUCKET` | 阿里云 OSS Bucket 名称 | `book-company-excel-uploads-v2` |
| `DINGTALK_WEBHOOK` | 钉钉机器人 Webhook 地址 | `https://oapi.dingtalk.com/robot/send?access_token=5b8ef7f0a7d203e858c5b00fdc99572081cf4813d1200aeed339bb217e9c7ad5` |
| `DINGTALK_SECRET` | 钉钉机器人加签密钥 | `SEC830ef80585c1e7cced0b4ef93068d55773fc271190859e98e2aec45c286e0cdc` |

### 配置步骤

1. 进入你的 Zeabur 项目页面
2. 点击你的服务（service）
3. 选择"环境变量"（Environment Variables）标签
4. 点击"添加环境变量"按钮
5. 逐一输入上述环境变量的键和值
6. 保存后，Zeabur 会自动重新部署你的服务

### 验证

部署完成后，访问 `https://你的域名.zeabur.app/upload_to_oss`，应该能看到批量上传页面。

## 二、阿里云函数计算（FC）环境变量配置

登录 [阿里云函数计算控制台](https://fcnext.console.aliyun.com/)，找到你的函数 `oss-excel-matcher`，在函数配置中添加环境变量。

### 必需的环境变量

| 环境变量名 | 说明 | 示例值 |
|---|---|---|
| `DINGTALK_WEBHOOK` | 钉钉机器人 Webhook 地址 | `https://oapi.dingtalk.com/robot/send?access_token=5b8ef7f0a7d203e858c5b00fdc99572081cf4813d1200aeed339bb217e9c7ad5` |
| `DINGTALK_SECRET` | 钉钉机器人加签密钥 | `SEC830ef80585c1e7cced0b4ef93068d55773fc271190859e98e2aec45c286e0cdc` |

> **注意**：FC 函数通过 RAM 角色访问 OSS，因此不需要配置 `OSS_ACCESS_KEY_ID` 和 `OSS_ACCESS_KEY_SECRET`。

### 配置步骤

1. 登录阿里云函数计算控制台
2. 进入"函数列表"，找到 `oss-excel-matcher` 函数
3. 点击函数名称，进入函数详情页
4. 点击"配置"标签
5. 在"环境变量"区域，点击"编辑"
6. 添加上述两个环境变量：
   - 键：`DINGTALK_WEBHOOK`，值：`https://oapi.dingtalk.com/robot/send?access_token=5b8ef7f0a7d203e858c5b00fdc99572081cf4813d1200aeed339bb217e9c7ad5`
   - 键：`DINGTALK_SECRET`，值：`SEC830ef80585c1e7cced0b4ef93068d55773fc271190859e98e2aec45c286e0cdc`
7. 点击"保存"

### 验证

上传测试文件到 OSS `tasks/张/` 目录（例如 `玉环新表.xlsx` 和 `玉环旧表.xlsx`），等待 10-20 分钟后，钉钉群应该收到机器人的通知消息，@你的手机号（`17737759937`）。

## 三、安全提示

⚠️ **重要安全提示**：

1. **不要将 AccessKey 和 Secret 提交到 Git 仓库**。始终使用环境变量配置，不要写在代码里。
2. **定期轮换密钥**：建议每 3-6 个月更换一次 AccessKey 和钉钉机器人加签密钥。
3. **最小权限原则**：确保阿里云 RAM 角色和 AccessKey 只拥有必要的权限（OSS 读写、FC 执行）。
4. **监控使用情况**：定期检查阿里云和钉钉的使用日志，发现异常及时处理。

## 四、常见问题

### Q1: 如何获取阿里云 AccessKey ID 和 Secret？

1. 登录[阿里云控制台](https://homenew.console.aliyun.com/)
2. 点击右上角头像 → "AccessKey 管理"
3. 推荐使用"子用户 AccessKey"（RAM 用户），而非主账号 AccessKey
4. 创建新的 AccessKey 对，记录 AccessKey ID 和 AccessKey Secret

### Q2: 如何获取钉钉机器人 Webhook 和加签密钥？

1. 在钉钉群中，点击右上角"..."→"智能群助手"→"添加机器人"
2. 选择"自定义"机器人（通过 Webhook 接入）
3. 配置机器人名称和安全设置（**必须开启"加签"**）
4. 记录生成的 Webhook 地址和加签密钥（Secret）

### Q3: 为什么上传文件后没有收到钉钉通知？

可能原因：
1. 环境变量配置错误或未配置
2. 钉钉机器人加签计算失败（检查 Secret 是否正确）
3. 函数计算执行失败（查看 FC 函数日志）
4. 手机号映射不正确（目前硬编码在 `fc_handler.py` 的 `_send_dingtalk_notification` 函数中）
